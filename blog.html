Introducción

La Calculadora de Paneles Solares es una aplicación web diseñada para estimar el número de paneles solares necesarios para un hogar.

Flujo Básico de la Aplicación

Inicio → Ubicación → Mes → Consumo → Resultados

Flujo de la Aplicación

La aplicación sigue un proceso secuencial para recolectar los datos necesarios:

1. Página de Inicio (index.html)

El archivo index.html es la página principal de la aplicación y cumple varias funciones clave:

Estructura Principal

  • Encabezado fijo: Contiene el título principal de la aplicación y se mantiene visible en todo momento.
  • Barra de navegación: Permite acceder a las diferentes secciones (Inicio, Ubicación, Información).
  • Sección hero: Área principal con llamados a acción y breve descripción de la aplicación.
  • Botón del blog: Enlace directo al blog técnico en la esquina superior izquierda.

Funcionalidades Clave

1. Diseño Atractivo:

  • Fondo con imagen temática y overlay oscuro para mejor legibilidad
  • Texto con sombras para mejor contraste
  • Botones con efectos hover interactivos

2. Navegación Intuitiva:

  • Botón "Comenzar Cálculo" inicia el flujo principal
  • Botón "Más información" lleva a detalles adicionales
  • Menú superior permite moverse entre secciones

3. Tecnologías Implementadas:

  • Diseño responsive que se adapta a móviles y tablets
  • Uso de localStorage para mantener datos entre páginas
  • Efectos CSS modernos para mejor experiencia de usuario

Flujo de Trabajo

  1. El usuario llega a la página de inicio
  2. Puede elegir entre:
    • Comenzar el cálculo (lo lleva a mapa.html)
    • Consultar información adicional (informacion.html)
    • Acceder al blog técnico
  3. El sistema verifica si hay datos previos almacenados
  4. Muestra la interfaz principal con los llamados a acción

Código Destacado

El JavaScript incluido realiza estas funciones:

  • Verifica si hay datos de radiación solar almacenados
  • Configura los event listeners para los botones principales
  • Previene el comportamiento por defecto de los enlaces para un control más preciso

Notas de Diseño: La página utiliza una estructura semántica HTML5 con secciones claramente definidas, manteniendo un equilibrio entre estética y funcionalidad. Los colores siguen la paleta principal de la aplicación (#2c3e50, #16a085, #ffffff) para consistencia visual.

2. Selección de Ubicación (mapa.html)

La página mapa.html es el primer paso en el flujo de cálculo, donde los usuarios seleccionan su ubicación para obtener datos precisos de radiación solar.

Componentes principales

1. Mapa Interactivo:

  • Implementado con Leaflet.js, una biblioteca JavaScript para mapas interactivos
  • Centrado inicialmente en México con zoom nivel 5
  • Capas de mosaicos de OpenStreetMap para la visualización
  • Marcador arrastrable para ajustar la ubicación exacta

2. Selector de Ciudades:

  • Lista desplegable con más de 30 ciudades importantes de México
  • Cada opción incluye coordenadas geográficas (latitud/longitud)
  • Selección automática de la ciudad más cercana al hacer clic en el mapa

3. Datos de Radiación:

  • Muestra tres tipos de radiación solar: Global (GHI), Directa (DNI) y Difusa (DHI)
  • Actualización en tiempo real al seleccionar nueva ubicación
  • Indicador de fecha/hora de la última actualización

Funcionamiento técnico

  1. Inicialización:
    • Carga el mapa con vista inicial de México
    • Configura los controladores de eventos para clics y selección de ciudad
  2. Interacción del usuario:
    • Al hacer clic en el mapa: coloca marcador y busca ciudad más cercana
    • Al seleccionar ciudad: centra el mapa y coloca marcador
  3. Obtención de datos:
    • Consulta la API de Open-Meteo para datos actuales de radiación
    • Parámetros solicitados: shortwave_radiation, direct_radiation, diffuse_radiation
    • Manejo de errores para fallos en la conexión
  4. Almacenamiento:
    • Guarda ubicación y datos en localStorage para uso en pasos siguientes
    • Datos almacenados: coordenadas, valores de radiación, ciudad seleccionada

Tecnologías clave

Leaflet.js

Biblioteca ligera para mapas interactivos, utiliza OpenStreetMap como proveedor de mosaicos base.

API Open-Meteo

Servicio meteorológico gratuito que proporciona datos actuales e históricos de radiación solar.

LocalStorage

Almacenamiento en el navegador para persistir datos entre páginas sin necesidad de servidor.

Flujo de datos

1. Usuario selecciona ubicación → 2. Sistema obtiene coordenadas → 3. Consulta API → 4. Procesa respuesta → 5. Actualiza interfaz → 6. Almacena datos → 7. Habilita siguiente paso

Consideraciones de diseño

  • Interfaz dividida en dos columnas (mapa y controles) para mejor usabilidad
  • Estados visuales claros para: carga, éxito y error en obtención de datos
  • Diseño responsive que se adapta a móviles (mapa encima, controles debajo)
  • Feedback inmediato al usuario con animaciones sutiles

3. Selección de Mes (mes.html)

La página mes.html permite seleccionar el periodo del año para el cálculo, lo que afecta directamente los resultados de radiación solar.

Componentes principales

1. Selector de Mes:

  • Lista desplegable con los 12 meses del año
  • Selección predeterminada en julio (mes con mayor radiación en México)
  • Actualización automática al cambiar la selección

2. Panel de Datos Climáticos:

  • Muestra radiación solar promedio diaria (kWh/m²)
  • Porcentaje estimado de días nublados
  • Radiación máxima registrada
  • Descripción contextual según los valores

3. Integración con NASA POWER API:

  • Consulta datos históricos de radiación solar
  • Manejo de errores con datos de ejemplo cuando la API falla
  • Fuente de datos claramente indicada

Funcionamiento técnico

  1. Carga inicial:
    • Obtiene ubicación guardada del localStorage
    • Carga datos del mes predeterminado (julio)
    • Muestra spinner durante la carga
  2. Cambio de mes:
    • Detecta cambio en el selector
    • Vuelve a consultar la API con nuevos parámetros
    • Actualiza la interfaz con los nuevos datos
  3. Obtención de datos:
    • Intenta conectar con NASA POWER API
    • Usa proxy CORS como respaldo
    • Si falla, genera datos estimados basados en latitud y mes
  4. Preparación para siguiente paso:
    • Guarda todos los datos en localStorage
    • Habilita el botón "Siguiente"

Tecnologías clave

NASA POWER API

Fuente de datos meteorológicos históricos de la NASA, proporciona información precisa de radiación solar.

Algoritmo de estimación

Cuando la API falla, genera datos basados en latitud y mes con patrones climáticos típicos.

LocalStorage

Almacena los datos climáticos para usarlos en los cálculos posteriores.

Flujo de datos

            1. Usuario selecciona mes → 
            2. Sistema construye consulta API → 
            3. Intenta conexión directa → 
            4. Si falla, usa proxy CORS → 
            5. Si falla, genera datos estimados → 
            6. Procesa y muestra resultados → 
            7. Almacena datos para siguiente paso

Consideraciones importantes

  • Resiliencia: Funciona incluso sin conexión a la API gracias a los datos estimados
  • Transparencia: Indica claramente cuando los datos son estimados
  • Experiencia de usuario: Feedback visual durante carga (spinner)
  • Precisión: Los datos reales de NASA POWER tienen resolución diaria

Ejemplo de consulta API

// Consulta a NASA POWER API para julio en Morelia
https://power.larc.nasa.gov/api/temporal/daily/point?
parameters=ALLSKY_SFC_SW_DWN,CLOUD_AMT&
community=RE&
longitude=-101.1844&
latitude=19.7008&
start=20230701&
end=20230731&
format=JSON

4. Consumo Eléctrico (consumo.html)

Esta página permite al usuario ingresar su patrón de consumo eléctrico, que es fundamental para calcular el sistema solar necesario.

Características principales

1. Métodos de ingreso flexibles:

  • Consumo promedio: Slider interactivo para seleccionar kWh/día
  • Personalizado: Opción avanzada para mayor precisión
  • Consumo horario: Tabla detallada por horas del día

2. Interfaz intuitiva:

  • Diseño responsive que se adapta a todos los dispositivos
  • Transiciones suaves entre diferentes métodos de ingreso
  • Validación automática de los valores ingresados

3. Integración con el sistema:

  • Almacena los datos en localStorage para el cálculo final
  • Calcula automáticamente el consumo diario total
  • Prepara los datos para la página de resultados

Funcionamiento técnico

  1. Carga inicial:
    • Muestra el método de ingreso predeterminado (consumo promedio)
    • Inicializa el slider con valor medio (5 kWh/día)
    • Prepara la tabla de consumo horario (24 filas)
  2. Interacción del usuario:
    • Al cambiar el método de ingreso, muestra/oculta las secciones correspondientes
    • Actualiza en tiempo real los valores mostrados
    • Valida que los valores ingresados sean números positivos
  3. Procesamiento de datos:
    • Para consumo promedio: toma directamente el valor del slider
    • Para consumo personalizado: suma los valores horarios o toma el valor diario
    • Guarda los datos en formato estructurado
  4. Preparación para resultados:
    • Combina los datos de consumo con ubicación y mes seleccionados
    • Almacena todo en localStorage bajo 'solarSimulationData'

Estructura de datos almacenados

// Ejemplo de datos guardados en localStorage
{
  "method": "personalizado",
  "dailyKwh": 8.5,
  "hourly": [0.1, 0.1, 0.1, ..., 0.8, 1.2, ...]
}

Validaciones importantes

  • Asegura que todos los valores sean números válidos
  • Impide valores negativos en el consumo
  • Proporciona valores por defecto razonables
  • Calcula automáticamente totales para verificar consistencia

Consideraciones de diseño

  • Jerarquía visual: Paso 3 claramente identificado
  • Progreso: Botones de navegación Anterior/Siguiente
  • Accesibilidad: Labels asociados a todos los controles
  • Feedback: Valores que se actualizan en tiempo real

5. Página de Resultados (resultados.html)

La página resultados.html es donde se consolidan todos los datos recolectados y se presentan los cálculos finales del sistema solar recomendado.

Estructura Principal

  • Resumen inicial: Muestra ubicación, consumo y número de paneles recomendados
  • Resultados principales: Explicación detallada de la cobertura estimada
  • Gráficos interactivos: Visualización de radiación vs consumo y distribución de energía
  • Escenarios comparativos: Diferentes configuraciones de paneles con sus pros y contras

Procesos de Cálculo

1. Cálculo de paneles necesarios:

  • Considera radiación solar promedio del lugar
  • Eficiencia típica de paneles (18%)
  • Área estándar por panel (1.6 m²)
  • Factor de pérdidas por temperatura/suciedad (75%)
  • Aplica 10% adicional como margen de seguridad

2. Modelado de variabilidad:

  • Simula 30 días de radiación considerando días nublados
  • Genera patrones de consumo con distribución normal
  • Clasifica días en excedentes, suficientes y con déficit

3. Escenarios comparativos:

  • Configuración mínima (menos paneles, menor inversión)
  • Configuración recomendada (balance óptimo)
  • Configuración ampliada (más paneles, excedentes)

Fórmulas Clave

Cálculo básico de paneles:

Paneles = (Consumo diario) / (Radiación × 0.18 × 1.6 × 0.75) × 1.1

Producción diaria estimada:

Producción = N° paneles × Radiación × 0.18 × 1.6 × 0.75

Porcentaje de cobertura:

Cobertura = (Producción diaria / Consumo diario) × 100

Flujo de Trabajo

  1. Recupera todos los datos almacenados (ubicación, mes, consumo)
  2. Calcula número base de paneles necesarios
  3. Genera simulaciones de 30 días considerando variabilidad
  4. Clasifica días según cobertura energética
  5. Prepara escenarios con diferentes cantidades de paneles
  6. Renderiza gráficos y resúmenes visuales

Tecnologías Clave

  • Chart.js: Para gráficos interactivos de radiación y consumo
  • LocalStorage: Para recuperar todos los datos ingresados
  • Modelado estadístico: Simulación de días nublados y patrones de consumo

Notas de Diseño: La página utiliza un diseño limpio con tarjetas informativas para presentar los resultados. Los gráficos proporcionan visión inmediata de los patrones, mientras que los escenarios permiten comparación fácil. La paleta de colores mantiene consistencia con el resto de la aplicación, usando verde para indicar excedentes y rojo para déficits.

Conclusión

El proyecto de la Calculadora de Paneles Solares representa una solución tecnológica accesible para estimar las necesidades de energía solar en hogares, combinando conocimientos técnicos con desarrollo web moderno.

Logros Principales

  • Interfaz intuitiva: Flujo guiado en 4 pasos simples (Ubicación, Mes, Consumo, Resultados)
  • Precisión técnica: Cálculos basados en datos reales de radiación solar y modelos energéticos
  • Flexibilidad: Adaptable a diferentes patrones de consumo y ubicaciones geográficas
  • Tecnología accesible: Implementado completamente con estándares web (HTML5, CSS3, JavaScript)

Impacto Potencial

  • Facilita la transición a energías renovables al hacer más comprensible la tecnología solar
  • Reduce la barrera de entrada para usuarios sin conocimientos técnicos
  • Proporciona estimaciones personalizadas sin necesidad de consultores especializados

Reflexión final: Este proyecto demuestra cómo herramientas tecnológicas sencillas pueden tener un impacto significativo en la promoción de energías limpias. La calculadora sigue evolucionando, con planes para incorporar más funcionalidades que ayuden a los usuarios a tomar decisiones informadas sobre su transición energética.
